## NVM & HDD

1. **基于闪存的NVM**经常用于类似磁盘驱动器的容器中，此时被称作固态硬盘，使用USB驱动器或DRAM stick的形式，就是U盘/USB移动闪存盘，做成DDR内存条外形，走内存通道就是闪存形态的NVM内存条。

2. 早年的主流是存储是**机械硬盘HDD**，统一连接到**SATA/IDE**接口，早期的SSD也是做成**SATA接口**，这些都是通过**接口/控制器**的间接连接，后来为了让SSD的性能跑满，推出**NVMe协议+PCLe**直连的M.2 SSD，让SSD的性能充分发挥。

3. SSD无法直接覆写，必须先擦除，而且必须**整块**擦（NAND 按页读取，空页写，整块擦），SSD**控制器**维护一个FTL（Flash Translation Layer）将请求中的逻辑块转化为实际的物理页，并且记录无效页，如果发现整个物理块都是无效页，SSD会在空闲时进行整块擦除。

4. **RAM驱动器**，驱动程序划分出DRAM的一部分当作硬盘来用： 
	<ul>
		<li> 裸块设备（底层读写，不常见）</li>
		<li> 格式化建立文件系统（拖文件，装缓存，解压临时文件）</li>
	</ul>
	为什么要格式化创建文件系统？fs是格式化出来的存储组织结构，OS中的**驱动**给了OS识别和读写这种结构的能力，OS想在这快DRAM使用执行文件操作就必须在其上建立fs，LinuxFHS中/tmp就是这种使用的体现，因此尽量在/tmp下工作，不仅速度在线，而且断电自动清空。

5. 硬盘需要先分区，OS会将每个分区逻辑上看作不同的物理设备，因此可以为不同的分区创建独立的文件系统，分区工具如Windows: diskmgmt.msc, DiskPart; Linux: fdisk, parted; PE。经典布局：EFI启动分区（100MB左右）用来引导开机，系统分区C盘装Windows内核+程序，交换分区/保留分区以及用户数据分区D盘都是可选的。

6. 分区后对不同区进行**格式化**，OS才能进行文件操作，格式化就是让原本的一个个存储单元变得有组织。

7. 要从系统分区中加载OS，硬盘中必须有**引导分区+引导程序+固件寻址地址**，引导分区就是FAT32格式的EFI，也叫系统分区ESP，在安装系统时自动创建这个小分区。现代：UEFI+GPT，老式：BIOS+MBR。
<ul>
<li> UEFI+GPT </li>
<ol>
	<li> 主板 **UEFI 固件**通电自检 → 识别硬盘 </li>
	<li> UEFI 自带分区识别能力，自动找到硬盘里的EFI ESP 引导分区</li>
	<li> 读取 ESP 里的官方引导程序（比如 Windows：`bootx64.efi`）</li>
	<li> 运行这个 `.efi` 引导代码（这个程序可以识别文件系统） → 再去找到 C 盘系统分区里的 **内核文件**（Windows `ntoskrnl.exe` / Linux 内核）</li>
	<li> 加载内核 → 启动完整操作系统 </li>
</ol>
<li> BIOS+MBR </li>
<ol>
<li> 硬盘 0 扇区是 **MBR 主引导记录** </li>
<li> BIOS 通电只读硬盘**第一个扇区**，执行里面的短小引导代码 </li>
<li> MBR 代码再找磁盘里标记为「活动 / 可引导」的分区 </li>
<li> 加载该分区分区引导记录 → 再启动 OS 内核 </li>
</ol>
</ul>

  8. 卷（volume)的创建和管理。有时，此步骤是隐式的，就像将文件系统直接放置在分区中一样，然后就可以挂载和使用卷。在其他时候，卷的创建和管理是显示的，例如，当多个分区或设备一起用作RAID集的时，一个或多个文件系统可以分布这些设备上。

## OS device driver 

1. 主机有一个设备控制器，设备自身也有一个控制器。主机控制器在**主板/CPU侧**，比如SATA控制器，NVMe PCIe控制器，USB主机控制器；设备控制器在**外设**里，比如SSD主控，U盘主控；他们中间是传输总线（SATA/NVMe/USB）。*控制器是协议的实现者*（Interface/Protocol），主机控制器生成符合协议格式的数据包，设备控制器解析并执行。总线定义**电气，机械和时序特性**不同的总线承载着不同的上层接口协议，根据协议标准在总线上传输消息，消息是格式化的命令数据包，解析格式依靠的就是**接口协议标准**，如NVMe命令，SATA ATA命令，USB U盘的Bulk传输命令。

2. OS的设备驱动分为**底层主机控制器驱动**专门控制*主机控制器*来初始化总线，收发底层消息，物理传输调度，和**上层设备类驱动/协议类驱动**面向具体外设（SSD/U盘），组装业务命令消息，交给**主机控制器驱动**去发送。所以OS设备驱动不直接驱动**设备控制器**（他们在外设里，想直接驱动也*摸不到*），OS驱动控制本机的**主机控制器**通过向**总线**发送**命令消息**间接指挥远端的**设备驱动器**干活。
	> 具体来说，比如OS要读写SSD，OS文件系统发出请求，**NVMe协议驱动**，也就是上层驱动，构造NVMe标准命令帧（Admin Command/I/O Command)，它只负责构造合法的NVMe消息并通过调用接口来调用PCIe**主机控制器驱动**，也就是底层驱动，由PCIe驱动完成PCIe寻址，DMA，物理链路收发数据，最终到达**SSD设备控制器**

3. 曾经OS调度磁盘需要知道硬件物理结构，读写要写上磁道，柱面，扇区，而且因为HDD的速度很慢，因此OS需要实现各种**磁盘调度算法**来优化磁盘访问，如FCFS，SSTF，SCAN，SCAN-r。现代的OS访问大容量存储统一使用LBA逻辑地址，由设备自己的控制器来转化为实际物理地址，比如SSD靠FTL，HDD靠内部寻址重映射。现代OS磁盘调度的目标为**公平性，及时性，整体IO吞吐优化**。
